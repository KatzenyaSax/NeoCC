### 问题2: 中文乱码

**现象**: 页面中文内容显示为乱码

**原因**: Nginx返回的Content-Type缺少charset=utf-8

**响应头**:
```
Content-Type: text/html
```

**应该是**:
```
Content-Type: text/html; charset=utf-8
```

## 🔍 问题分析

经过排查，发现**实际上不是真正的404错误**：

1. **页面可以访问**: `curl http://localhost/sales/customer` 返回200
2. **返回的是前端HTML**: 正确返回了Vue应用的index.html
3. **Vue Router工作正常**: `try_files $uri $uri/ /index.html` 配置正确

**结论**: 页面本身没有问题，可能是前端JavaScript加载或API调用问题。

### 乱码问题分析

**根本原因**: Nginx配置文件路径错误

**问题链路**:
1. 修改了 `ruoyi-ui/nginx.conf`（❌ 错误的位置）
2. 但docker-compose.yml挂载的是 `./nginx.conf`（✅ 正确的位置）
3. 导致修改的配置没有生效

**Docker配置**:
```yaml
nginx:
  volumes:
    - ./nginx.conf:/etc/nginx/nginx.conf:ro  # 使用的是根目录的nginx.conf
```

## ✅ 解决方案

### 修复1: 中文乱码问题

**文件**: `/Users/liuhongyu/IdeaProjects/final/NeoCC/nginx.conf`（根目录）

**修改内容**:
```nginx
# 修改前
location / {
    root /usr/share/nginx/html;
    index index.html index.htm;
    try_files $uri $uri/ /index.html;
    # 禁用缓存，防止旧版本 JS 被缓存
    add_header Cache-Control 'no-store, no-cache, must-revalidate, proxy-revalidate, max-age=0';
    add_header Pragma 'no-cache';
    expires off;
}

# 修改后
location / {
    root /usr/share/nginx/html;
    index index.html index.htm;
    try_files $uri $uri/ /index.html;
    # 设置正确的字符编码，防止中文乱码
    add_header Content-Type 'text/html; charset=utf-8';
    # 禁用缓存，防止旧版本 JS 被缓存
    add_header Cache-Control 'no-store, no-cache, must-revalidate, proxy-revalidate, max-age=0';
    add_header Pragma 'no-cache';
    expires off;
}
```

**重启Nginx**:
```bash
docker-compose restart nginx
```

**验证结果**:
```bash
$ curl -v http://localhost/
< Content-Type: text/html
< Content-Type: text/html; charset=utf-8  ✅
```

✅ **乱码问题已解决！**

- 应该修改 `./nginx.conf`（被docker-compose挂载）

### 中文字符编码配置

**Nginx配置**:
```nginx
location / {
    # 方式1: 使用charset指令（部分Nginx版本不支持）
    charset utf-8;
    
    # 方式2: 使用add_header强制设置（推荐）
    add_header Content-Type 'text/html; charset=utf-8';
}
```

**HTML META标签**:
```html
<meta charset="utf-8">
```

**双重保障**: Nginx响应头 + HTML META标签

### 前端路由404排查

**排查步骤**:
1. 使用curl测试真实HTTP状态码
2. 检查Nginx的try_files配置
3. 检查Vue Router配置（history模式）
4. 检查浏览器控制台JavaScript错误
5. 清除浏览器缓存

**常见误区**:
- 浏览器显示404，但curl返回200 → 浏览器缓存问题
- 页面加载了但内容空白 → JavaScript错误或API调用失败
- 页面样式错乱 → CSS文件加载失败或版本不匹配
