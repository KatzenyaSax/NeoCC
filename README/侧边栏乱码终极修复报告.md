## 🔍 根本原因分析

### 问题1: 数据库菜单数据双重UTF-8编码

**症状**:
```
数据库中: C3A7C2B3C2BB... (双重编码)
应该为:   E7B3BBE7BB9F... (正确UTF-8)
```

**原因**:
- 数据初始化时使用了错误的字符集
- UTF-8字节被再次当作Latin-1编码，然后重新编码为UTF-8
- 导致"系统管理"变成乱码

**验证**:
```sql
-- 错误的编码
SELECT HEX(menu_name) FROM sys_menu WHERE menu_id=1;
-- 返回: C3A7C2B3C2BB... ❌

-- 正确的编码
-- 应该返回: E7B3BBE7BB9F... ✅
```

**之前的问题**:
- `permission.js`中缺少`router.addRoute()`调用
- 导致动态路由未注册到Vue Router

**已修复**: ✅ 在之前的会话中已修复

## 💡 经验总结

### 1. 数据库字符集问题

**常见陷阱**:
- 数据库连接URL设置了`characterEncoding=utf8`
- 但客户端工具（如mysql命令行）没有指定字符集
- 导致INSERT/UPDATE时数据被错误编码

**正确做法**:
```bash
# 命令行连接时指定字符集
mysql --default-character-set=utf8mb4 -u root -p

# JDBC连接URL
jdbc:mysql://host:3306/db?characterEncoding=utf8&useUnicode=true
```

### 2. 双重UTF-8编码识别

**症状**:
- 显示乱码如 `ç³»ç»Ÿç®¡ç`
- HEX值以`C3`开头（UTF-8二次编码的特征）

**验证方法**:
```python
# Python验证
title = 'ç³»ç»Ÿç®¡ç'
# 错误的字节
print(title.encode('utf-8'))  
# b'\\xc3\\xa7\\xc2\\xb3...'

# 正确的"系统管理"字节应该是
print('系统管理'.encode('utf-8'))
# b'\\xe7\\xb3\\xbb\\xe7\\xbb\\x9f...'
```

### 3. Spring Boot多环境配置

**重要**:
- `application.yml` - 默认配置
- `application-docker.yml` - Docker环境配置
- 通过`SPRING_PROFILES_ACTIVE=docker`激活

**必须同步修改**:
- 如果修改了`application.yml`
- 必须同时修改`application-docker.yml`
- 否则Docker环境不会生效

### 4. Maven打包与Docker构建

**关键点**:
- Maven `clean package` 会将`src/main/resources`打包进JAR
- Dockerfile只复制JAR，不会复制外部配置文件
- **修改配置文件后必须重新Maven构建**

**正确流程**:
```bash
# 1. 修改配置文件
vim application-docker.yml

# 2. Maven重新打包
mvn clean package -DskipTests

# 3. Docker重新构建
docker-compose up -d --build service-name
```
