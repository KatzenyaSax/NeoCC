## ❌ 当前问题

### Redis连接失败

**错误信息**:
```json
{
    "code": 500,
    "message": "系统异常: Unable to connect to Redis",
    "data": null
}
```

**问题分析**:

1. ✅ Redis服务正常运行：`redis-cli ping` 返回 `PONG`
2. ✅ Redis配置正确：`host: neocc-redis, port: 6379`
3. ✅ Lettuce连接池已配置
4. ❌ **Spring Boot 3.x Lettuce客户端连接失败**

**可能原因**:

1. **Lettuce版本兼容性问题** - Spring Boot 3.2.5与Lettuce可能存在兼容性问题
2. **Docker网络DNS解析** - Auth容器可能无法解析`neocc-redis`主机名
3. **Redis连接超时** - 需要更长的超时时间或重试机制

---

## 🔧 解决方案

### 方案1: 切换到Jedis客户端（推荐）

Lettuce在Docker环境中有时不稳定，可以切换到Jedis：

**步骤1**: 修改pom.xml

```xml
<!-- auth/pom.xml -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
    <exclusions>
        <exclusion>
            <groupId>io.lettuce</groupId>
            <artifactId>lettuce-core</artifactId>
        </exclusion>
    </exclusions>
</dependency>
<dependency>
    <groupId>redis.clients</groupId>
    <artifactId>jedis</artifactId>
</dependency>
```

**步骤2**: 修改application-docker.yml

```yaml
spring:
  redis:
    host: neocc-redis
    port: 6379
    jedis:
      pool:
        max-active: 8
        max-wait: -1
        max-idle: 8
        min-idle: 0
```

### 方案2: 使用IP地址代替主机名

**修改application-docker.yml**:

```yaml
spring:
  redis:
    host: 172.18.0.3  # Redis容器的实际IP
    port: 6379
```

获取Redis容器IP：
```bash
docker inspect neocc-redis | grep IPAddress
```

### 方案3: 添加Redis连接诊断

在CaptchaController中添加连接测试：

```java
@GetMapping("/redis-test")
public Result<String> testRedis() {
    try {
        String pong = redisTemplate.getConnectionFactory()
            .getConnection()
            .ping();
        return Result.success("Redis连接成功: " + pong);
    } catch (Exception e) {
        return Result.error(500, "Redis连接失败: " + e.getMessage());
    }
}
```
