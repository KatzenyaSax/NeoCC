## ❌ 失败的测试

### 1. 验证码功能

#### 错误信息

```json
{
    "code": 500,
    "message": "系统异常: Failed to resolve 'neocc-auth' [A(1)]",
    "data": null
}
```

#### 根本原因

**Auth服务启动失败**，日志显示：

```
Caused by: java.lang.IllegalStateException: 
Ambiguous mapping. Cannot map 'ruoyiAdapterController' method
com.dafuweng.auth.controller.RuoyiAdapterController#captchaImage()
to {GET [/captchaImage]}: There is already 'captchaController' bean method
com.dafuweng.auth.controller.CaptchaController#captchaImage() mapped.
```

**问题分析**：
1. 代码层面已修复（删除了RuoyiAdapterController中的captchaImage方法）
2. 但Docker容器内使用的是**旧的JAR文件**（target目录中的编译产物）
3. Docker构建时执行 `COPY target/auth-*.jar app.jar`，拷贝的是旧文件

#### 解决方案

需要先在本地重新编译Auth服务：

```bash
# 方案1: 编译所有模块
cd /Users/liuhongyu/IdeaProjects/final/NeoCC
mvn clean package -DskipTests

# 方案2: 仅编译Auth模块
cd auth
mvn clean package -DskipTests

# 然后重新构建Docker镜像
cd ..
docker-compose up -d --build auth-service
```
