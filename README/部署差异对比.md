# NeoCC 项目部署配置说明

## 概述

本文档记录 NeoCC 项目的完整部署配置，包括 Docker Compose 基础设施、Gateway 路由、微服务配置及前后端联调方案。

**当前部署架构**：基于 Docker Compose + Nacos 服务发现 + MySQL 8 + Redis 7 的完整微服务环境

## 二、AuthFilter 认证过滤器

### 2.1 白名单配置

```java
// 问题：只放行了 /auth/** 路径
if (path.contains("/auth/api/sysUser/login") || path.contains("/auth/api/sysUser/page")) {
    return chain.filter(exchange);
}
```

### 2.2 白名单说明

```java
// 修复：添加 /api/** 和 /system/** 白名单
// 跳过登录/公开路径
if (path.contains("/auth/api/sysUser/login") || path.contains("/auth/api/sysUser/page")) {
    return chain.filter(exchange);
}

// 跳过 RuoYi 前端公开接口
if (PUBLIC_PATHS.contains(path)) {
    return chain.filter(exchange);
}

// 跳过业务 API 路径（/api/**）
if (path.startsWith("/api/")) {
    return chain.filter(exchange);
}

// 跳过系统管理路径（/system/**）
if (path.startsWith("/system/")) {
    return chain.filter(exchange);
}
```

### 2.3 差异说明

| 路径类别 | 示例 | 说明 |
|------|---------|------|
| 登录公开接口 | `/captchaImage`, `/login` | RuoYi 前端登录无需认证 |
| 业务 API | `/api/customer/**` | 所有业务接口暂放行 |
| 系统管理 | `/system/user/**` | RuoYi 系统管理接口 |
| 适配控制器 | `/sales/customer/**` | RuoYi 风格路由 |

## 三、微服务配置

### 3.1 基础设施（Docker Compose）

| 服务 | 容器名 | 端口 | 说明 |
|------|--------|------|------|
| Nacos | neocc-nacos | 8848, 9848 | 服务注册中心（Standalone） |
| MySQL | neocc-mysql | 3306 | 数据库（4 个独立库） |
| Redis | neocc-redis | 6379 | 缓存服务 |
| Gateway | neocc-gateway | 8086 | API 网关 |
| Auth | neocc-auth | 8085 | 认证授权服务 |
| System | neocc-system | 8082 | 系统管理服务 |
| Sales | neocc-sales | 8083 | 销售业务服务 |
| Finance | neocc-finance | 8084 | 财务业务服务 |
| Nginx | neocc-nginx | 80 | 前端静态资源托管 |

### 3.2 服务配置要点

**Nacos 配置**：
- 模式：Standalone（单机）
- 数据持久化：MySQL（dafuweng_nacos 库）
- 认证：已关闭（`NACOS_AUTH_ENABLE=false`）

**数据库配置**：
- 垂直拆分：4 个独立库（auth, system, sales, finance）
- 连接方式：Docker 容器名（neocc-mysql:3306）
- 健康检查：mysqladmin ping

```json
  "message": "操作成功",
  "data": {...}

```

前端 request.js 已适配处理 `res.data.msg` 或 `res.data.message`。

## 六、Nginx 配置差异

### 6.1 原始设计（GitHub）

无 Nginx 配置，前端直接使用开发服务器。

### 6.2 修复后配置（实际部署）

```nginx
http {
    server {
        listen 80;
        server_name localhost;

        # 前端静态资源
        location / {
            root /usr/share/nginx/html;
            index index.html index.htm;
            try_files $uri $uri/ /index.html;
            # 禁用缓存
            add_header Cache-Control 'no-store, no-cache, must-revalidate';
        }

        # API 代理到 Gateway
        location /prod-api/ {
            proxy_pass http://neocc-gateway:8086/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        location /dev-api/ {
            proxy_pass http://neocc-gateway:8086/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
}
```

## 七、部署启动流程

### 7.1 一键启动

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f gateway-service
```

### 7.2 服务启动顺序

1. MySQL（健康检查通过后继续）
2. Nacos（依赖 MySQL）
3. Redis
4. Auth / System / Sales / Finance（并行启动）
5. Gateway（等待所有微服务）
6. Nginx（最后启动）

### 7.3 访问地址

| 服务 | URL | 说明 |
|------|-----|------|
| 前端 | http://localhost | RuoYi-Vue3 管理后台 |

| Nacos | http://localhost:8848/nacos | 服务注册中心 |
| MySQL | localhost:3306 | 数据库 |
| Redis | localhost:6379 | 缓存 |

## 八、总结与建议

### 8.1 当前架构优势

1. ✅ **完整微服务架构**：Nacos 服务发现 + Gateway 统一路由
2. ✅ **前后端完全分离**：RuoYi-Vue3 + Spring Cloud 微服务
3. ✅ **数据库垂直拆分**：4 个独立库，故障隔离
4. ✅ **Docker 一键部署**：完整的 docker-compose 编排
5. ✅ **业务功能完整**：销售、财务、系统管理全覆盖

### 8.2 待优化项

1. **服务发现统一**：部分服务仍使用直接 URL，建议全部迁移到 Nacos
2. **认证强化**：业务 API 暂放行，后续需启用 JWT Token 验证
3. **监控完善**：接入 Spring Boot Actuator 实现服务监控
4. **日志集中**：引入 ELK 或 Loki 实现日志聚合

### 8.3 最佳实践

1. 数据库字段设计时考虑 NULL 约束和默认值
2. MyBatis-Plus 实体类显式使用 `@TableField` 注解
3. Gateway 路由优先使用 Nacos 服务发现
4. 前端 API 调用统一使用 axios 封装
5. 所有配置外部化，支持多环境切换
