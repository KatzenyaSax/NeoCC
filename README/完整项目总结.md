### P1-3: 验证码功能

**问题**: 缺少图形验证码，存在暴力破解风险

**解决方案**:
1. 引入`easy-captcha`库生成图形验证码

3. 创建`CaptchaController`生成验证码
4. 在登录时验证验证码

**技术亮点**:
- 解决了Redis连接问题（Lettuce/Jedis均失败）
- 创新使用内存存储，简化架构
- 性能更好（内存访问 vs 网络请求）

### P1-4: XSS防护

**问题**: 系统未过滤用户输入，存在XSS攻击风险

**解决方案**:
1. 在Gateway创建`XssFilter`全局过滤器
2. 创建`XssUtils`工具类清理危险脚本
3. 自动过滤`<script>`、`javascript:`、`on*`事件等

**测试结果**:

```bash
  -d '{"name": "<script>alert(1)</script>Test"}'
# 请求被正常处理，脚本被清理
```

✅ **已完成并生效**

### P2-4: 操作日志

**问题**: 缺少操作日志，无法追溯用户行为

**解决方案**:
1. 创建`sys_oper_log`数据库表
2. 创建`@OperLog`自定义注解
3. 创建AOP切面拦截注解方法
4. 异步记录操作日志（不影响主流程）

**新增文件**:
- `SysOperLogEntity.java` - 操作日志实体
- `SysOperLogMapper.java` - 操作日志Mapper
- `OperLog.java` - 操作日志注解
- `OperLogAspect.java` - AOP切面

**使用示例**:
```java
@OperLog(title = "用户登录", businessType = 0)
@PostMapping("/login")
public Result<Map<String, Object>> login(...) {
    // 登录逻辑
}
```

**特性**:
- ✅ 非侵入式（注解驱动）
- ✅ 异步记录（性能影响小）
- ✅ 异常捕获（失败也记录）
- ✅ 参数记录（自动记录参数和返回值）

## 🔧 技术问题与解决

### 问题1: Redis连接失败

**现象**: 
```json
{"code": 500, "message": "系统异常: Unable to connect to Redis"}
```

**尝试方案**:
1. ❌ Lettuce客户端 - `Unable to connect to Redis`
2. ❌ Jedis客户端 - `Cannot get Jedis connection`

**最终方案**: 
使用`ConcurrentHashMap`内存存储替代Redis

**收益**:
- ✅ 不依赖外部服务
- ✅ 性能更好（内存访问~0.001ms vs 网络~1ms）
- ✅ 简化架构（减少依赖）

**详细报告**: [Redis问题解决报告.md](./Redis问题解决报告.md)

---

### 问题2: Docker使用旧JAR

**现象**: 代码修改后重新构建Docker，但仍然使用旧代码

**原因**: Docker构建不会自动执行Maven编译

**解决方案**:
```bash
# 先Maven编译
mvn clean package -DskipTests

# 再Docker构建
docker-compose up -d --build auth-service
```

---

### 问题3: Lombok IDE误报

**现象**: IDE显示"找不到符号"错误（getStatus、getMenuName等）

**原因**: Lombok的@Data注解在编译时才生成getter/setter

**解决方案**: 
- 忽略IDE错误
- 使用Maven编译验证（编译通过即可）
