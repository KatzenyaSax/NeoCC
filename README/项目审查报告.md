#### 🔴 P0 问题：认证过滤器白名单放行

**文件**: `gateway/src/main/java/com/dafuweng/gateway/filter/AuthFilter.java`

**问题描述**:
```java
// 当前实现：所有业务 API 均放行，无需 Token
if (path.startsWith("/api/")) {
    return chain.filter(exchange);  // ❌ 安全风险
}

if (path.startsWith("/system/")) {
    return chain.filter(exchange);  // ❌ 安全风险
}
```

**影响**:
- ❌ 所有业务接口无需认证即可访问
- ❌ 数据权限控制失效
- ❌ 不符合企业级安全标准

**建议修复方案**（记录，暂不修改）:
```java
// 正确实现：验证 JWT Token
String token = exchange.getRequest().getHeaders().getFirst("Authorization");
if (token == null || !token.startsWith("Bearer ")) {
    exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
    return exchange.getResponse().setComplete();
}

// 调用 Auth 服务验证 Token
// 将 userId, roles, permissions 放入 Header 传递给下游服务
```

**优先级**: P0（安全关键）  
**预计工作量**: 2 天  
**依赖**: 需要 Auth 服务提供 Token 验证接口

## 四、安全性审查

### 4.1 认证与授权

#### 🔴 P0 问题：业务 API 无认证

**影响范围**: 所有 `/api/**` 和 `/system/**` 接口

**当前状态**:
```java
// AuthFilter.java - 白名单放行
if (path.startsWith("/api/")) {
    return chain.filter(exchange);  // ❌ 无需 Token
}
```

**风险**:
- ❌ 未登录用户可访问所有业务数据
- ❌ 无法追踪操作人
- ❌ 数据权限控制失效

**修复建议**（记录，暂不修改）:
1. 实现 JWT Token 签发（Auth 服务）
2. Gateway 验证 Token 有效性
3. 将用户信息放入 Header 传递给下游服务
4. 后端服务根据用户信息做数据权限过滤

**预计工作量**: 3-5 天
