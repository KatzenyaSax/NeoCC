## 🐛 发现的问题

### 问题1: 响应头缺少字符编码

**症状**:
- 中文显示为乱码: `è´¢åŠ¡ç®¡ç`
- Content-Type: `application/json`（缺少charset）

**根本原因**:
- Spring Boot默认不强制设置响应字符编码
- 后端服务未配置UTF-8编码
- Nginx未强制添加charset

**影响范围**:
- 所有API接口的中文内容
- 页面标题、按钮文字、数据字段

### 修复2: Nginx强制设置Content-Type

**文件**: `nginx.conf`

**修改位置**: `/dev-api/` 和 `/prod-api/` location块

**修改前**:
```nginx
location /prod-api/ {
    proxy_pass http://neocc-gateway:8086/;
    proxy_set_header Host $host;
    # ... 其他配置
}
```

**修改后**:
```nginx
location /prod-api/ {
    proxy_pass http://neocc-gateway:8086/;
    proxy_set_header Host $host;
    # ... 其他配置
    
    # 强制设置JSON响应的字符编码
    proxy_hide_header Content-Type;
    add_header Content-Type 'application/json; charset=utf-8' always;
}
```

**原理**:
1. `proxy_hide_header Content-Type` - 隐藏后端返回的Content-Type
2. `add_header Content-Type 'application/json; charset=utf-8' always` - 强制添加带charset的Content-Type

## 💡 经验总结

### 1. Spring Boot字符编码配置

**必须配置项**:
```yaml
server:
  servlet:
    encoding:
      charset: UTF-8
      force: true
      enabled: true

spring:
  http:
    encoding:
      charset: UTF-8
      force: true
      enabled: true
```

**说明**:
- `server.servlet.encoding` - Servlet容器的编码配置
- `spring.http.encoding` - Spring HTTP的编码配置
- `force: true` - 强制使用指定的编码
- `enabled: true` - 启用编码支持

### 2. Nginx强制设置响应头

**使用场景**:
- 后端服务无法修改时
- 需要统一设置响应头时
- 需要覆盖后端响应头时

**配置方法**:
```nginx
location /api/ {
    proxy_pass http://backend;
    
    # 隐藏后端响应头
    proxy_hide_header Content-Type;
    
    # 添加新的响应头
    add_header Content-Type 'application/json; charset=utf-8' always;
}
```
