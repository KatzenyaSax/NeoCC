## 🐛 问题描述

### 错误信息


### 影响

- ❌ 用户无法登录系统
- ❌ 无法获取动态路由
- ❌ 所有业务页面无法访问

---

## 🔍 问题分析

### 错误日志

Nginx错误日志显示：
```
connect() failed (111: Connection refused) while connecting to upstream
upstream: "http://172.18.0.9:8086/login"
```

### 根本原因

**Nginx缓存了Gateway的旧IP地址！**

**IP变化**:
- Nginx缓存的IP: `172.18.0.9` ❌
- Gateway实际IP: `172.18.0.10` ✅

**为什么会变化？**

Docker容器重启后，IP地址可能会改变：
1. Gateway服务重建或重启
2. Docker分配了新的IP地址
3. Nginx仍然使用旧的IP（DNS缓存）
4. 连接失败，返回404

---

## ✅ 解决方案

### 修复方法: 重启Nginx

```bash
docker-compose restart nginx
```

**为什么有效？**

重启Nginx会：
1. 清除DNS缓存
2. 重新解析`neocc-gateway`服务名
3. 获取Gateway的最新IP地址
4. 恢复正常代理

## 💡 预防措施

### 问题根源

Docker Compose服务重启后，容器IP可能变化：

```
第一次启动:
neocc-gateway: 172.18.0.9

重建后:
neocc-gateway: 172.18.0.10  ← IP变了！
```

### 解决方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **重启Nginx** | 简单快速 | 需要手动操作 |
| 使用resolver指令 | 自动刷新DNS | 配置复杂 |
| 固定IP | IP不变 | 不够灵活 |

### 推荐方案: 添加DNS resolver配置

**Nginx配置优化**（可选）:

```nginx
http {
    # 添加Docker DNS服务器
    resolver 127.0.0.11 valid=10s;
    
    server {
        location /prod-api/ {
            # 使用变量延迟解析
            set $gateway_backend http://neocc-gateway:8086;
            proxy_pass $gateway_backend;
            # ...
        }
    }
}
```

**优点**:
- Nginx每10秒刷新一次DNS
- 容器IP变化后自动适应
- 无需手动重启

**当前方案**:
- 简单有效
- 适合开发和测试环境
- 重启Nginx即可

## 🔧 排查流程

如果将来遇到类似问题，按以下步骤排查：

### 1. 检查错误日志

```bash
docker logs neocc-nginx 2>&1 | grep "error"
```

查找：
- `Connection refused` - 连接被拒绝
- `upstream` - 上游服务器地址
- `connect() failed` - 连接失败

### 2. 验证Gateway服务

```bash
# 检查Gateway是否运行
docker ps | grep gateway

# 检查Gateway IP
docker inspect neocc-gateway --format='{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'

# 直接测试Gateway
docker exec neocc-gateway curl -s -o /dev/null -w "%{http_code}" http://localhost:8086/login
```

### 3. 验证Nginx到Gateway的连接

```bash
# 从Nginx容器测试Gateway
docker exec neocc-nginx curl -s -o /dev/null -w "%{http_code}" http://neocc-gateway:8086/login
```

### 4. 检查DNS解析

```bash
# 查看Nginx解析的Gateway IP
docker exec neocc-nginx cat /etc/resolv.conf

# 手动解析服务名
docker exec neocc-nginx getent hosts neocc-gateway
```

### 5. 重启Nginx

```bash
docker-compose restart nginx
```

## 🎯 经验总结

### 1. Docker容器IP是动态的

**重要认知**:
- Docker容器的IP地址不是固定的
- 容器重建后IP可能变化
- 服务名（DNS）是可靠的，IP不可靠

### 2. Nginx会缓存DNS解析结果

**默认行为**:
- Nginx启动时解析一次DNS
- 之后使用缓存的IP
- 不会自动刷新

**解决方案**:
- 重启Nginx（简单）
- 配置resolver指令（自动）

### 3. 排查网络问题的步骤

```
1. 查看错误日志
   ↓
2. 确认服务是否运行
   ↓
3. 检查IP地址是否匹配
   ↓
4. 测试服务间连通性
   ↓
5. 重启相关服务
```

### 4. Docker Compose环境下的最佳实践

**开发环境**:
```bash
# 服务重建后，重启Nginx
docker-compose up -d --build gateway-service
docker-compose restart nginx
```

**生产环境**:
```nginx
# 配置自动DNS刷新
resolver 127.0.0.11 valid=30s;
set $backend http://service-name:port;
proxy_pass $backend;
```
