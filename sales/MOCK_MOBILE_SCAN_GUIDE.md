## 📝 使用流程

### 完整的扫码登录流程

#### 第 1 步：Web 端生成二维码

1. 访问 `http://localhost/login`
2. 点击"扫码登录"
3. 页面显示二维码
4. 按 F12 打开控制台，复制 loginTid

或者使用测试页面：
1. 访问 `http://localhost/qrcode-test.html`
2. 点击"生成二维码"
3. 复制显示的 loginTid

---

#### 第 2 步：打开手机端（模拟）

1. 访问 `http://localhost/qrcode-admin.html`
2. 页面显示三个输入框：
   - **Login TID**：从二维码解析的内容
   - **用户名**：当前登录的用户名
   - **密码**：当前登录的密码

---

#### 第 3 步：输入信息

**方式一：手动输入**
```
Login TID：粘贴从 Web 端复制的 loginTid
用户名：admin
密码：admin123
```

**方式二：模拟扫码**
```
（在实际应用中，这里会调用摄像头扫码）
（现在手动输入 loginTid 即可）
```

---

#### 第 4 步：确认登录

1. 点击 **"✅ 确认登录"** 按钮
2. 系统会自动执行以下步骤：
   - ✅ 使用用户名密码登录，获取 userId
   - ✅ 调用扫码接口（后端会验证用户）
   - ✅ 确认登录
   - ✅ Web 端自动登录

3. 查看消息提示：
   - ✅ 绿色：成功
   - ❌ 红色：失败

---

## 🔐 安全验证流程

### 后端验证逻辑

```
手机端提交：
{
  "loginTid": "xxx",
  "username": "admin",
  "password": "admin123"
}
         ↓
步骤 1：验证用户名密码
    调用 /login 接口
    获取 userId
         ↓
步骤 2：扫码
    调用 /api/oauth2/qrcode/scan
    后端从数据库验证用户
    ✅ 检查用户是否存在
    ✅ 检查用户状态
    ✅ 使用数据库真实用户名
         ↓
步骤 3：确认登录
    调用 /api/oauth2/qrcode/confirm
    更新状态为 CONFIRMED
         ↓
步骤 4：Web 端获取 Token
    Web 端轮询检测到确认
    获取 JWT Token
    自动登录
```

## 🔍 工作原理

### API 调用流程

```javascript
// 步骤 1: 登录获取 userId
POST /prod-api/login
{
  "username": "admin",
  "password": "admin123"
}

响应：
{
  "code": 200,
  "data": {
    "userId": 1
  }
}

// 步骤 2: 扫码
POST /prod-api/api/oauth2/qrcode/scan
{
  "loginTid": "abc123",
  "userId": 1,  // 从步骤 1 获取
  "username": "admin",
  "deviceId": "mobile-app"
}

// 步骤 3: 确认登录
POST /prod-api/api/oauth2/qrcode/confirm
{
  "loginTid": "abc123",
  "userId": 1,
  "confirm": true
}
```
