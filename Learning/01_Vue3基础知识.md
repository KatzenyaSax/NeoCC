## 1. 什么是 Vue？

### 1.1 官方定义

Vue (读音 /vjuː/，类似 view) 是一个**渐进式 JavaScript 框架**，用于构建用户界面。

### 1.2 通俗理解

想象你正在用乐高积木搭建一个房子：

- **HTML** = 积木块（结构）
- **CSS** = 积木的颜色和样式（外观）
- **JavaScript** = 让积木动起来的指令（行为）
- **Vue** = 帮你更快、更方便地组装积木的**说明书和工具箱**

### 1.3 Vue 能做什么？

- ✅ 构建单页面应用（SPA）
- ✅ 开发企业级后台管理系统
- ✅ 移动端 H5 页面
- ✅ 小程序（uni-app 基于 Vue）
- ✅ 桌面应用（Electron + Vue）

## 2. 为什么使用 Vue？

### 2.1 相比原生 JS 的优势

**不用 Vue（原生 JavaScript）：**
```javascript
// 当用户输入变化时，更新页面显示
document.getElementById('input').addEventListener('input', function(e) {
    document.getElementById('display').innerText = e.target.value
})
```

**用 Vue：**
```vue
<!-- 只需要写一行代码，Vue 自动帮你同步 -->
<input v-model="message">
<p>{{ message }}</p>
```

### 2.2 Vue 的优势

| 特性 | 说明 |
|------|------|
| **上手简单** | 文档友好，学习曲线平缓 |
| **响应式** | 数据变化，页面自动更新 |
| **组件化** | 把页面拆成独立可复用的积木 |
| **生态丰富** | 有配套的路由、状态管理、UI库 |
| **社区活跃** | 遇到问题容易找到解决方案 |

## 3. Vue 3 的核心概念

### 3.1 什么是 Composition API？

Vue 3 引入了 Composition API（组合式 API），这是一种新的编写组件逻辑的方式。

### 3.2 对比 Options API vs Composition API

**Options API（Vue 2 风格）：**
```vue
<script>
export default {
  // 数据
  data() {
    return {
      count: 0,
      name: '张三'
    }
  },
  // 方法
  methods: {
    addCount() {
      this.count++
    }
  },
  // 计算属性
  computed: {
    doubleCount() {
      return this.count * 2
    }
  },
  // 生命周期
  mounted() {
    console.log('组件挂载完成')
  }
}
</script>
```

**Composition API（Vue 3 推荐）：**
```vue
<script setup>
// 导入需要的函数
import { ref, computed, onMounted } from 'vue'

// 数据
const count = ref(0)  // ref 用于创建响应式数据
const name = ref('张三')

// 方法
function addCount() {
  count.value++  // 注意：修改 ref 需要用 .value
}

// 计算属性
const doubleCount = computed(() => count.value * 2)

// 生命周期
onMounted(() => {
  console.log('组件挂载完成')
})
</script>
```

### 3.3 为什么推荐 Composition API？

1. **更好的逻辑复用** - 可以把相关逻辑抽离到独立的函数中
2. **更灵活的代码组织** - 相关代码可以放在一起，而不是分散在 data/methods/computed 中
3. **更好的 TypeScript 支持** - 对类型推导更友好
4. **更好的代码压缩** - 变量名更短，压缩后体积更小

## 4. 响应式数据

### 4.1 什么是响应式？

**响应式**就是：当数据发生变化时，页面自动更新。

就像 Excel 表格一样：
- 单元格 A1 = 10
- 单元格 B1 = A1 * 2
- 当 A1 变成 20 时，B1 自动变成 40

### 4.2 ref() - 基础类型响应式

**使用场景：** 基本类型数据（数字、字符串、布尔值）

```vue
<script setup>
import { ref } from 'vue'

// 定义响应式数据
const count = ref(0)        // 数字
const message = ref('你好')  // 字符串
const isActive = ref(true)   // 布尔值

// 修改数据（注意 .value）
function addCount() {
  count.value++  // count 变成 1
}

function updateMessage() {
  message.value = 'Hello Vue'  // message 变成 'Hello Vue'
}
</script>

<template>
  <div>
    <p>计数：{{ count }}</p>           <!-- 显示: 0 -->
    <p>消息：{{ message }}</p>          <!-- 显示: 你好 -->
    <p>状态：{{ isActive ? '激活' : '未激活' }}</p>
    
    <button @click="addCount">+1</button>
    <button @click="updateMessage">修改消息</button>
  </div>
</template>
```

### 4.3 reactive() - 对象类型响应式

**使用场景：** 对象和数组

```vue
<script setup>
import { reactive } from 'vue'

// 定义响应式对象
const user = reactive({
  name: '张三',
  age: 25,
  address: {
    city: '北京',
    district: '朝阳区'
  }
})

// 修改数据（直接用 . 访问，不需要 .value）
function updateUser() {
  user.name = '李四'
  user.age = 30
  user.address.city = '上海'
}

// 添加数组元素
const hobbyList = reactive(['篮球', '足球'])

function addHobby() {
  hobbyList.push('游泳')
}
</script>

<template>
  <div>
    <p>姓名：{{ user.name }}</p>
    <p>年龄：{{ user.age }}</p>
    <p>城市：{{ user.address.city }}</p>
    <p>爱好：{{ hobbyList }}</p>
    
    <button @click="updateUser">修改用户</button>
    <button @click="addHobby">添加爱好</button>
  </div>
</template>
```

### 4.4 ref vs reactive 对比

| 特性 | ref() | reactive() |
|------|-------|------------|
| 适用类型 | 基本类型 + 对象 | 只能用于对象/数组 |
| 访问方式 | 需要 `.value` | 直接访问 |
| 解构 | 可以解构，保持响应式 | 解构会失去响应式 |
| 重新赋值 | 可以 | 不可以（需整个替换） |

```vue
<script setup>
import { ref, reactive, toRefs } from 'vue'

// ref - 可以重新赋值
const count = ref(0)
count.value = 10  // ✅ 可以

// reactive - 不能重新赋值整个对象
const user = reactive({ name: '张三' })
// user = reactive({ name: '李四' })  // ❌ 错误！

// 正确做法：修改属性
user.name = '李四'  // ✅ 可以

// reactive 解构问题
const state = reactive({ count: 0, name: '张三' })

// ❌ 错误：解构后失去响应式
let { count, name } = state

// ✅ 正确：使用 toRefs 保持响应式
let { count, name } = toRefs(state)
</script>
```

## 5. 条件渲染与循环

### 5.1 条件渲染 v-if / v-show

```vue
<script setup>
import { ref } from 'vue'

const isLoggedIn = ref(true)
const role = ref('admin')

function toggleLogin() {
  isLoggedIn.value = !isLoggedIn.value
}
</script>

<template>
  <div>
    <!-- v-if：条件为 false 时，元素从 DOM 中移除 -->
    <div v-if="isLoggedIn">
      <p>欢迎回来！</p>
    </div>
    <div v-else>
      <p>请登录</p>
    </div>

    <!-- v-else-if 和 v-else -->
    <div v-if="role === 'admin'">
      <p>管理员面板</p>
    </div>
    <div v-else-if="role === 'user'">
      <p>用户面板</p>
    </div>
    <div v-else>
      <p>访客面板</p>
    </div>

    <!-- v-show：条件为 false 时，元素隐藏但保留在 DOM 中 -->
    <div v-show="isLoggedIn">
      <p>显示隐藏示例</p>
    </div>

    <button @click="toggleLogin">切换登录状态</button>
  </div>
</template>
```

**v-if vs v-show 区别：**

| 特性 | v-if | v-show |
|------|------|--------|
| 原理 | 条件渲染（真正的添加/删除 DOM） | CSS 显示/隐藏（display: none） |
| 切换开销 | 高（每次切换都要重建 DOM） | 低（只是切换 CSS） |
| 初始开销 | 低（条件为 false 时不渲染） | 高（始终渲染，只是隐藏） |
| 适用场景 | 很少切换的内容 | 频繁切换的内容 |

### 5.2 循环渲染 v-for

```vue
<script setup>
import { ref, reactive } from 'vue'

// 数组
const fruits = ref(['苹果', '香蕉', '橙子'])

// 对象数组
const users = reactive([
  { id: 1, name: '张三', age: 25 },
  { id: 2, name: '李四', age: 30 },
  { id: 3, name: '王五', age: 28 }
])

// 对象
const person = reactive({
  name: '张三',
  age: 25,
  city: '北京'
})

function addFruit() {
  fruits.value.push('葡萄')
}

function removeFruit(index) {
  fruits.value.splice(index, 1)
}
</script>

<template>
  <div>
    <!-- 遍历数组 -->
    <h3>水果列表</h3>
    <ul>
      <li v-for="(fruit, index) in fruits" :key="index">
        {{ index + 1 }}. {{ fruit }}
        <button @click="removeFruit(index)">删除</button>
      </li>
    </ul>
    <button @click="addFruit">添加水果</button>

    <!-- 遍历对象数组 -->
    <h3>用户列表</h3>
    <table border="1">
      <thead>
        <tr>
          <th>ID</th>
          <th>姓名</th>
          <th>年龄</th>
        </tr>
      </thead>
      <tbody>
        <tr v-for="user in users" :key="user.id">
          <td>{{ user.id }}</td>
          <td>{{ user.name }}</td>
          <td>{{ user.age }}</td>
        </tr>
      </tbody>
    </table>

    <!-- 遍历对象 -->
    <h3>人物信息</h3>
    <ul>
      <li v-for="(value, key) in person" :key="key">
        {{ key }}: {{ value }}
      </li>
    </ul>
  </div>
</template>
```

### 5.3 :key 的重要性

`:key` 是 Vue 识别列表中每个元素的"身份证"，帮助 Vue 高效更新列表。

```vue
<!-- ✅ 推荐：使用唯一 ID 作为 key -->
<li v-for="user in users" :key="user.id">

<!-- ⚠️ 不推荐：使用数组索引作为 key（但数据不变时可以） -->
<li v-for="(user, index) in users" :key="index">
```

## 6. 事件处理

### 6.1 基本用法 @click

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)
const name = ref('Vue')

// 方法定义
function handleClick() {
  count.value++
  console.log('点击了！')
}

function handleClickWithParam(message) {
  console.log(message)
}

// 内联调用
function handleInline() {
  console.log('内联调用')
}
</script>

<template>
  <div>
    <p>计数：{{ count }}</p>
    
    <!-- 基本点击事件 -->
    <button @click="handleClick">点击 +1</button>
    
    <!-- 传递参数 -->
    <button @click="handleClickWithParam('你好')">带参数点击</button>
    
    <!-- 内联表达式 -->
    <button @click="count++">内联 +1</button>
    
    <!-- 获取事件对象 $event -->
    <button @click="handleClickWithParam($event)">获取事件对象</button>
  </div>
</template>
```

### 6.2 常用事件修饰符

```vue
<template>
  <div>
    <!-- .stop - 阻止冒泡（点击子元素时，不触发父元素的事件） -->
    <div @click="parentClick">
      <button @click.stop="childClick">点击我</button>
    </div>

    <!-- .prevent - 阻止默认行为（如表单提交、链接跳转） -->
    <form @submit.prevent="handleSubmit">
      <input v-model="inputValue">
      <button type="submit">提交</button>
    </form>

    <!-- .enter - 按下回车键触发 -->
    <input @keyup.enter="handleEnter" placeholder="按回车提交">

    <!-- 组合修饰符 -->
    <button @click.stop.prevent="handleClick">组合使用</button>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const inputValue = ref('')

function parentClick() {
  console.log('父元素被点击')
}

function childClick() {
  console.log('子元素被点击')
}

function handleSubmit() {
  console.log('表单提交:', inputValue.value)
}

function handleEnter() {
  console.log('按下了回车')
}
</script>
</template>
```

### 6.3 常用事件列表

| 修饰符 | 说明 |
|--------|------|
| `.stop` | 阻止事件冒泡 |
| `.prevent` | 阻止默认行为 |
| `.capture` | 使用捕获模式 |
| `.self` | 只有自身触发 |
| `.enter` | 回车键 |
| `.esc` | ESC 键 |
| `.space` | 空格键 |

## 7. 计算属性与方法

### 7.1 计算属性 computed

**计算属性**：基于已有的响应式数据，计算出一个新的值。

```vue
<script setup>
import { ref, computed } from 'vue'

const firstName = ref('张')
const lastName = ref('三')
const count = ref(5)

// 基础用法
const fullName = computed(() => {
  return firstName.value + lastName.value
})

// 可写计算属性
const fullName2 = computed({
  get() {
    return firstName.value + lastName.value
  },
  set(value) {
    // 当 fullName2.value = '李 四' 时触发
    const names = value.split(' ')
    firstName.value = names[0]
    lastName.value = names[1] || ''
  }
})

// 计算属性缓存
const doubleCount = computed(() => {
  console.log('计算属性执行了')  // 依赖不变时不重复执行
  return count.value * 2
})
</script>

<template>
  <div>
    <p>姓：<input v-model="firstName"></p>
    <p>名：<input v-model="lastName"></p>
    <p>姓名：{{ fullName }}</p>
    
    <button @click="count++">count +1</button>
    <p>count × 2 = {{ doubleCount }}</p>
  </div>
</template>
```

### 7.2 方法 methods

**方法**：每次调用都会重新执行函数。

```vue
<script setup>
import { ref } from 'vue'

const count = ref(5)

// 方法
function doubleCount() {
  console.log('方法执行了')  // 每次调用都执行
  return count.value * 2
}
</script>

<template>
  <div>
    <button @click="count++">count +1</button>
    <!-- 每次组件重新渲染都会重新调用 -->
    <p>{{ doubleCount() }}</p>
    <p>{{ doubleCount() }}</p>
  </div>
</template>
```

### 7.3 computed vs methods 区别

| 特性 | computed | methods |
|------|----------|---------|
| 缓存 | 有缓存，依赖不变不重复计算 | 无缓存，每次调用都执行 |
| 性能 | 更高（避免重复计算） | 较低 |
| 使用方式 | 当属性使用 `{{ fullName }}` | 当函数调用 `{{ getName() }}` |
| 适用场景 | 计算派生数据 | 需要执行逻辑操作 |

```vue
<!-- computed 有缓存 -->
<p>{{ fullName }}</p>
<p>{{ fullName }}</p>
<!-- fullName 只计算一次 -->

<!-- methods 无缓存 -->
<p>{{ getName() }}</p>
<p>{{ getName() }}</p>
<!-- getName 执行两次 -->
```

## 8. 生命周期

### 8.1 什么是生命周期？

生命周期就是 Vue 组件从创建到销毁的整个过程。在这个过程中，Vue 会在特定时刻自动调用一些函数。

### 8.2 生命周期图

```
┌─────────────────────────────────────────────────────────────┐
│                        组件实例创建                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  setup()  ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←│
│     │                                                         │
│     ▼                                                         │
│  onBeforeMount()  ← 模板编译完成，即将挂载到 DOM              │
│     │                                                         │
│     ▼                                                         │
│     │  ←←←←←←←←←←←←←←←←←←←←← DOM 挂载 ←←←←←←←←←←←←←←←←←←←←←←│
│     │                                                         │
│     ▼                                                         │
│  onMounted()  ← 组件已挂载到 DOM，可以访问 DOM 元素          │
│     │                                                         │
│     ▼                                                         │
│  onBeforeUpdate()  ← 数据变化，DOM 更新之前                  │
│     │                                                         │
│     ▼                                                         │
│  onUpdated()  ← DOM 更新完成                                 │
│     │                                                         │
│     ▼                                                         │
│  onBeforeUnmount()  ← 组件即将卸载                          │
│     │                                                         │
│     ▼                                                         │
│  onUnmounted()  ← 组件已卸载完成                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 8.3 常用生命周期钩子

```vue
<script setup>
import { ref, onMounted, onUpdated, onUnmounted } from 'vue'

const message = ref('Hello Vue')

// 组件挂载后（常用）
onMounted(() => {
  console.log('组件已挂载到 DOM')
  // 通常在这里：获取 DOM、发起请求、设置定时器
  const element = document.querySelector('.container')
  console.log(element)
})

// 数据更新后
onUpdated(() => {
  console.log('组件更新了')
  console.log('当前 message:', message.value)
})

// 组件卸载前（清理资源）
onUnmounted(() => {
  console.log('组件即将卸载')
  // 通常在这里：清除定时器、取消订阅、移除事件监听
})
</script>

<template>
  <div class="container">
    <p>{{ message }}</p>
    <button @click="message = 'Updated'">更新消息</button>
  </div>
</template>
```

### 8.4 实际使用场景

```vue
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

const count = ref(0)
let timer = null  // 定时器 ID

onMounted(() => {
  console.log('组件挂载，启动定时器')
  timer = setInterval(() => {
    count.value++
  }, 1000)
})

onUnmounted(() => {
  console.log('组件卸载，清除定时器')
  if (timer) {
    clearInterval(timer)
  }
})
</script>

<template>
  <p>计时器：{{ count }}</p>
</template>
```

## 9. 组件基础

### 9.1 什么是组件？

组件就是把页面拆分成独立的、可复用的积木块。

```
┌─────────────────────────────────────┐
│           App.vue (根组件)          │
│  ┌───────────┐  ┌───────────────┐  │
│  │ Header    │  │  Content       │  │
│  │ (头部组件) │  │ (内容组件)      │  │
│  └───────────┘  │  ┌───────────┐ │  │
│                 │  │ Card      │ │  │
│  ┌───────────┐  │  │ (卡片组件) │ │  │
│  │ Footer    │  │  └───────────┘ │  │
│  │ (底部组件) │  └───────────────┘  │
│  └───────────┘                      │
└─────────────────────────────────────┘
```

### 9.2 定义组件

**文件：`src/components/UserCard.vue`**

```vue
<template>
  <div class="user-card">
    <h3>{{ name }}</h3>
    <p>年龄：{{ age }}</p>
    <button @click="sayHello">打招呼</button>
  </div>
</template>

<script setup>
// 定义 props（父组件传递的数据）
const props = defineProps({
  name: {
    type: String,
    default: '匿名'
  },
  age: {
    type: Number,
    default: 0
  }
})

// 定义事件（子组件向父组件传递消息）
const emit = defineEmits(['greet'])

function sayHello() {
  emit('greet', `你好，我叫 ${props.name}！`)
}
```

### 9.4 Props 和 Emit 详解

**Props（父 → 子）：**
```vue
<!-- 静态传递 -->
<ChildComponent title="Hello" />

<!-- 动态传递（绑定变量） -->
<ChildComponent :title="myTitle" />

<!-- 传递数组/对象 -->
<ChildComponent :items="itemList" />
```

**Emit（子 → 父）：**
```vue
<!-- 子组件 -->
<script setup>
const emit = defineEmits(['click', 'update', 'delete'])

function handleClick() {
  emit('click', '传递的数据')
}
</script>

<!-- 父组件 -->
<ChildComponent @click="handleClick" />
```

### 10.2 一个完整的列表页面

**文件：`src/views/UserList.vue`**

```vue
<template>
  <div class="user-list">
    <!-- 搜索区域 -->
    <div class="search-bar">
      <input 
        v-model="searchName" 
        placeholder="输入用户名搜索"
        @keyup.enter="handleSearch"
      >
      <button @click="handleSearch">搜索</button>
      <button @click="handleReset">重置</button>
    </div>

    <!-- 用户列表 -->
    <div class="user-list">
      <div 
        v-for="user in filteredUsers" 
        :key="user.id" 
        class="user-card"
      >
        <h3>{{ user.name }}</h3>
        <p>邮箱：{{ user.email }}</p>
        <p>状态：{{ user.status === 1 ? '启用' : '禁用' }}</p>
        <button @click="handleEdit(user)">编辑</button>
        <button @click="handleDelete(user.id)">删除</button>
      </div>
    </div>

    <!-- 空状态 -->
    <div v-if="filteredUsers.length === 0" class="empty">
      <p>暂无数据</p>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, onMounted } from 'vue'

// 响应式数据
const searchName = ref('')
const userList = ref([])

// 计算属性：过滤后的用户列表
const filteredUsers = computed(() => {
  if (!searchName.value) {
    return userList.value
  }
  return userList.value.filter(user => 
    user.name.includes(searchName.value)
  )
})

// 方法
function handleSearch() {
  console.log('搜索:', searchName.value)
}

function handleReset() {
  searchName.value = ''
}

function handleEdit(user) {
  console.log('编辑用户:', user)
}

function handleDelete(id) {
  console.log('删除用户:', id)
}

// 生命周期
onMounted(() => {
  console.log('组件挂载，加载数据')
  // 模拟加载数据
  userList.value = [
    { id: 1, name: '张三', email: 'zhangsan@example.com', status: 1 },
    { id: 2, name: '李四', email: 'lisi@example.com', status: 0 },
    { id: 3, name: '王五', email: 'wangwu@example.com', status: 1 }
  ]
})
</script>

}
```

## 总结

### 核心要点回顾

| 概念 | 说明 |
|------|------|
| **ref()** | 创建基本类型的响应式数据 |
| **reactive()** | 创建对象类型的响应式数据 |
| **v-if / v-show** | 条件渲染 |
| **v-for** | 列表渲染 |
| **@click** | 事件处理 |
| **computed** | 计算属性（有缓存） |
| **methods** | 方法（无缓存） |
| **onMounted** | 生命周期钩子 |
